# 10k-per-section differential sweep vs the real DOS engine — 2026-06-29

Fresh round of randomized differential sweeps of the Go port against the real
DOS computational engine (`legacy/oracle/`, rebuilt for the Linux sandbox via
`build_linux.sh`), targeting **10,000 fuzzer cases per test section**.

## Headline

Across **~230,000 randomized cases** spanning amortization, mortgage, and
present-value, **zero financial-logic divergences**. Two cases tripped the
ultra-tight relative-error gates (1e-6 / 1e-4); both were investigated and are
**sub-cent floating-point noise on degenerate/extreme inputs that round to the
same cent** as DOS. Nothing to fix.

## The two near-tolerance flags — both benign

**1. Mortgage solve-monthly.** `price=1,196,396.14, 78.3% down, 12yr, 15.39%,
balloon=$411,953.90 at month 3`. The balloon ($411,954) exceeds the financed
principal (~$259,617), so the loan is retired almost immediately and the
residual "monthly payment" solves to **$0.87**. DOS=0.870312 vs Go=0.870311 —
**absolute difference $0.000001** (one-millionth of a dollar). Only trips the
1-ppm gate because the denominator (the payment) is near zero. Degenerate input.

**2. Amortization payment solve, prepaid variant.** `amount=$3,067.83, 5.853%,
295 monthly payments`. DOS=**$19.63720** vs Go=**$19.64000** — absolute
difference **$0.0028** (a quarter of a cent). Both round to the **same $19.64**
payment. Sub-cent solver-tail noise on a long, low-payment prepaid loan.

Every other section's worst-case relative error is ≤ ~9e-5 (well under a cent on
realistic payments).

## Relation to the $2 "AM Example 1" question

This sweep specifically hammers the odd-first-period / prepaid / balloon / basis
space that the $2 help-text-vs-DOS item (`discrepancies.md` §7) lives in. The
port tracks the DOS engine to the cent throughout — reinforcing that the §7 $2 is
a Windows-help artifact, not a port divergence.

## Reproducing

```
legacy/oracle/build_linux.sh                       # amort_oracle
TARGET=pv_oracle  legacy/oracle/build_linux.sh     # pv_oracle
TARGET=mtg_oracle legacy/oracle/build_linux.sh     # mtg_oracle

PERSENSE_FUZZ=1 PERSENSE_FUZZ_N=10000 \
  go test ./internal/finance/... -run 'Fuzz|Sweep|Cube|Baseline|Solve' -v
```

The three `zzfuzz_oracle_test.go` files (amort/mortgage/PV) were updated this
round to honor `PERSENSE_FUZZ_N` (they previously had hardcoded loop counts), and
their divergence prints now include the absolute-dollar difference so a future
near-tolerance flag is immediately classifiable as rounding vs. logic.
