Mortgage Help
Fill in Price or Monthly Total (not both). Fill in exactly one of: % Down, Cash Required, or Amt Borrowed — whichever you leave blank is what Per%Sense solves for (the fill-in-the-blank concept). Computed cells look like $1,234.56 — these are outputs, recalculated on every Calculate, so they aren't used as inputs. To reuse a computed value as a fixed input (and solve for a different blank), harden it: double-click the cell or press H. If you just want a different number, type over it instead. Click Clear Row to start over.
Shortcuts: C or F10 calculates the selected row; Enter moves to the next field (and recalculates if Auto-calculate is on); H hardens the selected computed cell.
More: Mortgage screen guide · the grid & balloons · APR & loan comparisons · What-If tables.

Amortization Help
Enter loan parameters below. Leave Payment Amount blank to compute it, or leave Amount Borrowed or Rate blank to back-solve it. Points are used only for APR — they do not affect the schedule.
Note: any Advanced Options in effect stay in effect until cleared (a red badge on the panel shows how many) — use Clear All before starting an unrelated loan so a leftover target, moratorium, or skip-month doesn't quietly change the schedule.
Shortcuts: C or F10 calculates; press T in any date field for today's date; Enter moves to the next field (and recalculates if Auto-calculate is on).
More: Amortization guide · Advanced Options · payoff / balance lookup · settings that affect the schedule.

Additional Periodic Payments ?Recurring extra payments, e.g. biannual extras. By default each is paid in addition to the regular payment; the "Balloon / prepayment includes regular pmt" setting can switch them to replace the regular payment instead. Learn more →

Rate/Payment Adjustments ?For adjustable-rate mortgages: change rate and/or payment on specific dates. Learn more →
Date ?Date the rate or payment change takes effect. Must be after Loan Date and before Last Pmt Date. New Rate % ?New annual interest rate effective from this date forward. Leave blank if only the payment amount is changing. New Amount ?New regular payment amount effective from this date forward. Leave blank if only the rate is changing — Per%Sense then re-amortizes the remaining balance at the new rate (DOS AO5). Leave the rate blank and supply only an amount to have the engine solve the implied rate (AO6). Leaving both blank is the AO7 "re-amortize at current rate" signal.

Present Value Help
Enter payments above, rate and as-of date below. Leave Rate blank to compute an IRR (internal rate of return). The as-of date can be past (present value) or future (future value).
Note: the help-doc worked examples are quoted using the True Rate convention — pick it in Rate Type to reproduce their figures. Every row and modifier listed below is summed into the total, so use Clear All before starting an unrelated example.

Variable Rate Schedule ?Use different interest rates for different time periods (e.g. IRS tax interest, statutory rates). When any rate is populated, the As-of Rate field above is ignored. IRR cannot be computed in variable rate mode. Learn more →
Enter rates that change over time. Row 1's rate is the starting rate (its date is fixed at "—"); subsequent rows specify when the rate changes. Fill in any one of True Rate, Loan Rate, or Yield per row — they're equivalent and the engine uses the True Rate form internally (LoanRate assumes monthly compounding).
Effective Date ?Date this rate takes effect. Row 1 is read-only (the starting rate); subsequent rows change the rate from that date forward. True Rate % ?Continuously compounded rate — the mathematically "pure" rate. Passed directly to the engine. Loan Rate % ?Nominal annual rate compounded monthly. Converted to True Rate via TrueRate = 12 × ln(1 + LoanRate/12). Yield % ?Effective annual yield. Converted to True Rate via TrueRate = ln(1 + Yield).
