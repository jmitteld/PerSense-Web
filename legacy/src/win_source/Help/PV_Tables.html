Generating Present Value Tables
The Present Value screen enables you to create tables of future payments and their value, as are often supplied by forensic experts. The table contains a line for each future payment, the amount of that payment, and its contribution to the present value computed in the bottom block.

You can generate present value tables here the same way you generatetables while working on the Amortization screen. To initiate table output, press Ctrl-T . Specify whether you want the table to go on the screen, to a file,or directly to the printer. Press Esc if you want to leave the table before it is finished printing.

Complete instructions for the Table Output window are on pages 60 - 62

Note: You might want to check your Printer Settings and Directories Settings (press Alt-S ) before you begin.

To obtain a present value table from your calculations, you must tell PerSense what level of detail you want and where you want to send it. Remember that anything sent to the screen will not be saved in any other format.

Note: Here the total value computed at the bottom of the table (648,362.68) is identical to the number computed on the screen, in the top Present Value line. At other times, you might find there is a slight discrepancy.

The discrepancy will appear if payments are monthly (or quarterly, etc) and the 365-day calendar is used, but not if payments are weekly or bi-weekly. In the real, 365-day calendar, months are of unequal length, and PerSense neglects this in order to calculate more quickly. By assuming that payments are spaced equally, the program is able to use a formula to compute the value of a whole set of payments at once, instead of summing the values individually.

In these cases, the answer at the bottom of the table is slightly more accurate than the answer on the screen. If you want the screen as well as the table to reflect this extra accuracy you can choose the Exact setting in the Computational Settings screen. Press Alt-S and select the Computational Settings screen, then specify YES for Exact mode. (You should also specify 365- day basis; if you are computing in the 360-day calendar, you have already made the approximation of equal months.)
