The Variable Rate Present Value Screen
The regular Present Value screen permits a great deal of flexibility in listing payments, but to reduce the payments to present value, a single interest rate is used. There are times when you may need to compute a present value using different interest rates for different time periods. PerSense Pro offers a screen with all the flexibility of the Present Value screen, and the added power of variable rates.

A common use for this feature is the computation of tax interest: the IRS specifies a table of rates that must be used to compute interest on past-due taxes. Different rates must be used during different time periods, within the same computation.

Another use for this feature is in civil law, where monetary damages in the past must be valued in the present. In some states, the interest rate to use for this computation is specified by statute, and varies over time. The Variable Rate Present Value screen is designed specifically for these situations.

The Present Value screen without Variable Rate is the right place to perform IRR calculations. Since by definition an IRR is a single interest rate that summarizes an entire transaction, the concept is a poor fit for the Variable Rate screen.

In the Variable Rate Present Value screen, interest rates in the table of rates cannot be the target of a computation.

After selecting the Present Value screen with Alt-P , you can toggle Variable Rate on and off with the letter V .

The top half of the Variable Rate screen works just as it does in the other Present Value screen. But the bottom block works on a different concept: With Variable Rate turned off, the Present Value block at bottom contains three independent lines. Each one is a complete summary of the value of all the payments above as of a certain date. But with Variable Rate turned on, this same area of the screen contains a rate schedule, with different interest rates active at different times.

The block at lower right contains a single summary of the present value of all payments listed above, using the rate schedule entered in the lower left.

Enter the schedule of interest rates in the lower left of the screen. At left, enter the date on which each rate takes effect. (The first date is entered for you as XX this is the interest rate that is in effect at the beginning of your computation.) Then you may specify each rate as a true rate, a loan rate, or a yield.

Like the regular Present Value screen, the Variable Rate screen supports forward computations of present value and backward computations of any payment amount. On the regular Present Value screen, you can also compute a rate (an IRR, for example), but in the Variable Rate screen, the Rate Table is read only you cannot make an interest rate the target on the Variable Rate screen.

In the bottom right, enter an As of date. The Variable Rate screen also cludes an option for SIMPLE interest. Simple interest should never be used in commercial problems. But for legal purposes, some states (unfairly) specify that a plaintiff may only sue for simple interest on his past losses. (These same states use compound interest to reduce future losses to a present value. The result is to minimize the value in both cases.)

If you are doing a computation involving simple interest on past losses and compound interest on future losses, you must separate the two computations, since the SIMPLE / COMPOUND option prevails through an entire computation.

The Present Value screen with Variable Rate turned off computes only compound interest.
